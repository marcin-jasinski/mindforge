# Personas — MindForge UI Redesign

**Phase**: 3 — Persona Exploration
**Approved**: 2026-04-29

---

## Persona 1: Alex — The Ambitious Self-Learner

| Attribute | Value |
|---|---|
| **Age** | 25–35 |
| **Role** | Full-time worker, self-directed learner |
| **Tech comfort** | High |
| **Primary device** | Desktop/laptop, evenings and weekends |

### Goals
- Absorb and retain knowledge from books, articles, and PDFs faster
- Build a personal knowledge base worth returning to
- Feel like studying is actually working (visible progress)

### Pain Points
- Gets frustrated when a tool feels like a chore to open — visual quality signals quality of the tool
- Loses streaks because the habit doesn't stick (no visible motivation triggers)
- Can't tell if they're actually making progress without digging through menus

### Key Journey
1. **Discovery**: Uploads a PDF or article to a Knowledge Base
2. **Active Learning**: Reviews flashcards daily — 10–15 minute sessions
3. **Testing**: Takes a quiz when feeling confident, uses the score as motivation
4. **Big Picture**: Checks the concept map occasionally to see how ideas connect
5. **Retention**: Returns weekly, streaks and "due today" count are the hook

### Design Implications
- Low-friction, calm, motivating aesthetic
- Subtle but visible progress signals (streak, due count in sidebar)
- The app should feel like a reward when opened, not homework
- Quick access to "Resume studying" from any screen

## Persona 2: Maya — The Certification Grinder

| Attribute | Value |
|---|---|
| **Age** | 28–40 |
| **Role** | IT/Finance professional, exam-driven learner |
| **Tech comfort** | Very high |
| **Primary device** | Desktop at work desk, sometimes laptop at home |

### Goals
- Pass a specific professional exam (AWS, CISSP, CFA, PMP, etc.)
- Retain dense, technical material under time pressure
- Trust that AI-generated questions match real exam difficulty
- Identify and close weak areas before exam day

### Pain Points
- Loses confidence in the tool when the UI looks unpolished ("if the UI is rough, is the AI reliable?")
- Needs to see what she still needs to study — coverage gaps matter
- Wants to show the tool to a colleague or manager without embarrassment

### Key Journey
1. **Setup**: Creates a separate KB per exam domain (e.g., "AWS Security", "Networking")
2. **Ingestion**: Uploads official study guides, whitepapers, practice dumps
3. **Study Loop**: Quiz → identify weak areas → Flashcards on weak concepts → repeat
4. **Verification**: Uses Search to clarify specific terms or compare definitions
5. **Concept Review**: Uses Concept Map to understand relationships before exam

### Design Implications
- Professional, credible aesthetic — looks impressive if shown to a colleague
- Clear progress tracking at the KB level (what's studied, what's pending)
- Concept Map must look modern and trustworthy — it's a proof-of-intelligence feature
- Dense information presented cleanly (table/list views for Documents page)
- Keyboard-first interactions (power user habits)
